# 建中社團線上點名系統 (CKHS Club Attendance System)

一個為建國中學學生社團設計的線上出缺席點名與簽名管理系統。前台基於 **HTML/CSS/JS (Vanilla)** 打造，後台完全運行於 **Google Apps Script (GAS)**，並以 **Google 試算表 (Google Sheets)** 與 **雲端硬碟 (Google Drive)** 作為資料儲存與報表管理中心。

link:https://rollcall.cksc.tw/

---

## 🌟 系統特色

### 1. 學生身分與權限隔離
* **Google 帳號整合登入**：串接 Google 一鍵登入 (Google Identity Services)，限制僅能使用學校官方信箱 (`@gl.ck.tp.edu.tw`) 進行驗證。
* **幹部專屬權限**：僅限社團幹部（如社長、副社長、教學等）可使用「開始點名」與「修改歷史紀錄」功能。
* **社員專屬查詢**：普通社員登入後只能查看自己個人的出缺席紀錄，無法瀏覽他人資訊或進行修改。
* **正副社長警告機制**：非正/副社長之幹部在開始點名或修改時，系統會彈出警告，提示其紀錄將同步學務處，防範越權操作。
* **外部測試登入**：提供測試帳號與密碼登入管道，方便校外指導老師或管理員進行測試。

### 2. 智慧化點名與簽名功能
* **自動帶入資訊**：登入後自動帶入當前社團名稱、填寫人，且鎖定不可修改，防範資料竄改。
* **無紙化手寫簽名板**：指導老師可直接在行動裝置或電腦螢幕上手寫簽名，系統將簽名轉換為 Base64 格式圖片並自動上傳至 Google Drive 保存。
* **草稿自動儲存 (localStorage)**：任何填寫內容、打勾狀態都會即時儲存在本機。若不慎關閉網頁，下次開啟表單時會自動提示是否還原草稿。
* **限時修改防呆**：點名紀錄僅允許在補填天數限制內（如 5 天內）進行修改，過期後自動鎖定，且日期選擇器限制只能選擇過去 5 天內的課堂。

### 3. 效能優化與快取機制 (Caching)
* **快取快速登入**：檢測到有效的快取 Token 時，瞬間導向至主選單，避免每次重新整理都卡在 Google 驗證等待畫面。
* **名單 10 分鐘 TTL 限制**：載入社團學生名單時，引入 10 分鐘效期快取，避免短時間內反覆進出表單造成的重覆 GAS 網路調用。
* **歷史清單背景刷新 (Stale-While-Revalidate)**：優先以快取顯示過往點名清單，並在背景非同步查詢最新紀錄，在完成後動態刷新，實現秒開查詢頁面的流暢體驗。

### 4. 試算表底層公式與報表彙整
* **當日缺曠自動彙整**：自動跨社團提取全校指定日期的缺席名單，自動 VLOOKUP 匹配學號，產出結構化名冊。
* **校內/校外老師出席統計**：自動計算全校指導老師請假與出席率，並把校內老師代課紀錄獨立列出。
* **自訂 Sheets 選單功能**：在 Google 試算表後台選單中可直接執行「產出指定日期簽名報表 (Google 文件)」以及「新學期清空資料」。
* **互動式查詢面板**：只要在試算表中選擇社團名稱，即可以矩陣圖表直接查閱全學期出缺席。

## 📁 檔案結構

```bash
├── index.html          # 登入與測試登入頁面
├── menu.html           # 主選單頁面 (導向點名或查詢)
├── records.html        # 出缺席歷史紀錄與詳細資訊頁面
├── form.html           # 點名填寫、簽名板與學生清單表單
├── style.css           # 系統全域樣式表 (含 RWD 手機版適配)
├── app.js              # 前端核心邏輯與 API 互動腳本
├── system-map.html     # 互動式系統架構圖 (說明頁面)
# --- Google Apps Script 後台程式碼 ---
├── form.js             # 後台 API 核心路由器與資料提交防護
├── sign.js             # 產生指定日期簽名 Google 文件報表
├── club_search.js      # 試算表「查詢面板」聯動繪製矩陣
├── setting.js          # 後台試算表自訂功能選單建立
└── clear.js            # 新學期清空資料與簽名檔功能
```

## ⚙️ 部署與安裝步驟

### 1. 建立後台試算表
1. 新增一份 Google 試算表。
2. 建立以下工作表（工作表名稱須完全一致）：
   * `原始名單`：欄位包含 `班級`、`座號`、`姓名`、`學號`、`身分`、`社團編號`、`社團名稱`。
   * `點名紀錄`：用於儲存前台送出的資料。
   * `系統設定`：在 `B2` 儲存簽名報表匯出的目標 Google Drive 資料夾 ID。
   * `原始社團`、`統計總表`、`當日缺曠`、`校內出席老師紀錄`、`校外出席老師紀錄`。

### 2. 部署 Google Apps Script
1. 在試算表上方選單點選 **擴充功能** -> **Apps Script**。
2. 將專案中的 `form.js`、`sign.js`、`club_search.js`、`setting.js`、`clear.js` 分別貼入 Apps Script 專案的對應 `.gs` 檔案中。
3. 點選右上方 **部署** -> **新增部署**：
   * 類型選擇 **網頁應用程式 (Web App)**。
   * 執行身分選擇 **具有存取權的使用者 (User accessing the web app)**。
   * 誰能存取選擇 **任何人 (Anyone)**。
4. 部署完成後複製產生的 **網頁應用程式網址 (Web App URL)**。

### 3. 設定前端網頁
1. 打開 `app.js`，將最上方的 `GAS_URL` 修改為您部署得到的 Web App URL：
   ```javascript
   const GAS_URL = 'https://script.google.com/macros/s/您的網頁應用程式網址/exec';
   ```
2. 將 `CLIENT_ID` 修改為您申請的 Google OAuth Client ID。
3. 將所有前端檔案 (`index.html`, `menu.html`, `records.html`, `form.html`, `style.css`, `app.js`) 上傳至靜態網頁託管平台（如 GitHub Pages、Vercel 或學校伺服器）即可上線運作。

## 🛡️ 安全防護機制
* **CORS 預檢繞過**：前端 `gasPost` 採用 `text/plain` 傳輸 JSON 內容，規避 Google Apps Script 無法響應 OPTIONS 預檢請求的問題。
* **冷卻時間防刷 (Cooldown)**：同一用戶 60 秒內禁止重複提交，防止連擊產生重複數據。
* **資料過濾 (Sanitization)**：後台寫入時自動清理字串控制字元、HTML 標籤，防範 XSS 攻擊；對於公式前置字元（如 `=`, `+`, `-`）自動加上零寬度字元防止試算表公式注入。
* **Token HMAC 簽章驗證**：內部測試帳號使用加密 HMAC-SHA256 Token 簽署身分，並設有過期機制。
