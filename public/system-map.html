系統總覽
整個系統由三大部分組成：使用者看到的前台網頁、處理邏輯的後台伺服器 (Google Apps Script)、以及儲存資料的 Google 雲端。

前台網頁

使用者直接操作的介面，包含登入、填寫點名表、簽名、查詢紀錄等畫面。
檔案：index.html + style.css + app.js

後台伺服器 (GAS)

Google Apps Script 部署在雲端，負責身分驗證、資料讀寫、安全防護。所有前台請求都經由它處理。
核心檔案：form.js

Google 雲端儲存

Google 試算表存放學生名單、點名紀錄；Google Drive 資料夾儲存老師手寫簽名圖檔與簽名報表。

安全機制

多層防護：Google Token 驗證、HMAC 加密簽章、60 秒冷卻防刷、XSS 防護、公式注入阻擋、權限隔離。

試算表結構 (6 張工作表)

原始名單：全校社團學生
點名紀錄：歷次點名資料
系統設定：資料夾路徑與測試帳號
統計總表：社團對照表
查詢面板：互動式報表
系統日誌：錯誤紀錄

使用者身分

幹部：可「開始點名」+「查詢紀錄」+「修改紀錄」
社員：只能查自己的出缺席
測試帳號：外部人員用密碼登入

系統運作概念
前台每一個操作，都會透過 HTTP POST 呼叫後台的 API 路由器。後台驗證身分後，對 Google 試算表進行讀取或寫入，再將結果回傳給前台顯示。
前後台溝通的完整路徑

▸
① 使用者在前台操作
例如：點擊「開始點名」、「確認送出」、「查詢紀錄」
② 前台呼叫 gasPost()
將操作指令與資料打包成 JSON，透過 HTTP POST 送到 Google Apps Script
③ 後台 doPost() 接收請求
解析 JSON，根據 action 欄位判斷要執行哪個功能
④ 身分驗證 authenticateRequest()
檢查 Google Token 或測試 Token 是否有效，確認使用者身分
⑤ 執行對應功能
讀取或寫入 Google 試算表、上傳簽名到 Google Drive
⑥ 回傳結果 JSON
將處理結果（成功/失敗/資料）回傳給前台
⑦ 前台更新畫面
根據回傳結果顯示成功訊息、錯誤提示、或渲染資料列表
